Q1：請用一句話說明什麼是「虛擬化」技術。
A1：「虛擬化」技術就是透過軟體層將實體硬體資源（如 CPU、記憶體、硬碟）進行抽象化與切割，讓一台實體電腦能同時獨立運行多個虛擬作業系統或應用程式。

Q2：你今天使用的練習環境涉及哪兩種虛擬化技術？請簡單說明它們各自的角色。
A2：硬體虛擬化 (Hardware Virtualization / Cloud VM)：

角色： 提供基礎的運算平台。你所連線的主機 IP（如 54.151.244.27）是運行在 AWS 或類似雲端平台上的 虛擬機器 (EC2 Instance)。它負責模擬出整套硬體環境（CPU、記憶體、磁碟），讓 Linux 作業系統能夠穩定運行在遠端資料中心的實體伺服器上。

作業系統層虛擬化 (OS-level Virtualization / Containers)：

角色： 提供多使用者隔離的練習環境。由於作業要求中提到「帳號分配表」且多位同學共用同一台主機 IP，這通常是透過 Docker 容器 或 Linux 多使用者隔離 技術來實現。它讓每位同學在同一個作業系統核心下，擁有獨立的工作目錄與權限，像是在各自的小隔間裡操作，互不干擾。

Q3：如果改用傳統虛擬機器來提供同樣的練習環境，你覺得會有什麼不同？
A3：傳統虛擬機器： 每個學生都要下載好幾 GB 的鏡像檔（如 ISO），並在自己電腦安裝 VirtualBox。優點是完全掌控，缺點是非常耗費本機資源（效能吃緊），且環境設定容易出錯。

目前的練習環境： 你只需透過 SSH 連線到雲端主機。優點是環境統一、不佔用本機效能，只要有網路就能隨時隨地練習。
